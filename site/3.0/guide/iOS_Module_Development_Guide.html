iOS Module_Development_Guide

개요

타이타늄은 타이타늄+플러스 모듈이라고 불리는 추가적인 확장을 통해 내장 기능을 확장할 수있는 방법을 제공한다. 이 문서에서 이런 모듈을 확장하는 방법을 간략하게 알아본다.

이 문서는 다음과 같은 iOS 모듈 개발 과정을 다룬다.

  • 1단계, iOS 모듈 개발 환경 설정 한다.
  • 2단계, 첫 예제 모듈을 만들기 위한 과정을 차근차근 밟아본다.
  • 3단계, 모듈의 기본 아키텍처를 설명한다.
  • 4단계, 앞에서 만든 모듈을 배포하는 방법에 대해 설명한다.

iOS 모듈 개발을 위한 준비

iOS용 모듈을 만드려면, iOS 애플리케이션을 만드는데 필요한 모든 소프트웨어가 다 필요하다.

  • 타이타늄 SDK
  • iOS SDK와 해당 버전에 맞는 Xcode, 제세한 내용은 iOS SDK 설치와 업데이트를 참고한다.
  • 타이타늄 스튜디오 또는 타이타늄 커멘드-라인 인터페이스(CLI)

추가로 모듈 빌드를 위해서 파이썬과 파이썬 마크다운(markdown) 모듈이 모두 필요하다. 자세한 설치방법은 다음을 참고한다.

  • 파이썬
  • 파이썬 설치툴

iOS 애플리케이션 개발과 마찬가지로 iOS 모듈을 개발하며려 맥 OS X가 반드시 필요하다.

1단계: 모듈 개발 환경 설정하기

개발 환경을 설정하려면 먼저 헬퍼 스크립트를 타이타늄 설치 경로에 두거나 기존 구버전의 타이타늄 CLI 스크립트(titanium.py)를 가리키는 별도의 이름을 설정해야한다. 참고로 베타 버전에 포함된 최신 타이타늄 CLI는 모듈 개발을 지원하지 않는다.

자세한 별칭 설정 방법은 레거시 커멘드-라인 인터페이스를 참고한다.

2단계: 첫번째 모듈 만들기

모듈을 만들기 전에 테스트 모듈을 만들어서 환경설정이 제대로 됐는지 확인해보자.

모듈을 만들기 위해서는 다음과 같은 명령을 콘솔창에 입력한다.

$titanium.py create --platform=iphone --type=module --dir=~/tmp/ --name=test --id=com.example.test

명령 실행후 다음과 같은 메시지를 볼수있다.

Appcelerator Titanium XCode templates installed Created iphone module project

이제 새로운 모듈은 ~/tmp/test 경로에 만들어졌을 것이다. 그럼 새로 생성된 모듈을 확인해보자.

$cd ~/tmp/test $ls -la

Filename / Directory        Description / Purpose
.gitignore                  A special .gitignore file used by the revision control system git to instruct it to ignore certain temporary files and directories. This file is not distributed with your module. This file can be safely ignored if you are not using git.
Classes                     The directory where you Objective-C headers and implementation classes should go. These files are not distributed with your module and are used by the Xcode compiler.
ComExampleTest_Prefix.pch   The pre-compiled header file. This file is not distributed with your module and is used by the Xcode compiler.
LICENSE                     The file for describing the contents of your module's license text. This file is distributed with your module to end-users.
README                      The file that gives a short explanation of the module project. This file is not distributed with your module.
assets                      The directory where you should place module assets such as images.
build.py                    This file is a script that will compile and package your module for usage and distribution.
documentation               The directory where you should place your module documentation for end-users. The file index.md is a Markdown-formatted template file that you should use when writing your module documentation. This is only required for module distributed in the Titanium+Plus Market-place. You can safely ignore this directory if you do not intend to distribute your module.
example                     The directory where your module example(s) should go. The file app.js will be generated to include a sample loading of your module in a test window. This file can be used for quickly testing your module as well as give an example to end-users on how to use your module. This directory is distributed with your module.
hooks                       The directory is currently ignored and reserved for future use.
manifest                    A special file that describes meta-data about your module and used by the Titanium compiler. This file is required and is distributed with your module.
module.xcconfig             A special file used by Xcode when your module is compiled in an end-user Titanium application which references your module. This file is a Xcode configuration file which can setup special compiler and linker directives that are specific to your module. This file is distributed with your module.
platform                    Directory for platform-specific assets. This is analogous to the platform directory in a standard Titanium application project, and contains platform-specific assets that are not processed by Titanium (for example, Android styles and themes).
test.xcodeproj              The directory which contains your Xcode project. This directory is not distributed with your module.
timodule.xml                Titanium module configuration file. The format is described in tiapp.xml and timodule.xml Reference.
titanium.xcconfig           A special file used by Xcode when your module is compiled when building your module. This file is a Xcode configuration file which can setup special compiler and linker directives that are specific to your module. This file is not distributed with your module and only used during module development.
